# 论文复现思路与代码实现完整梳理

## 论文目标

用深度学习方法还原被水下介质（悬浮物、光线散射）破坏的偏色图像，使其恢复为自然色彩。
整个项目分为 **4 大核心模块**，代码对应 4 个 `.py` 文件。

---

## 第一步：数据准备 — `dataset.py`

### 做了什么？
告诉模型"这张图长什么样子"，以及"它应该被修成什么样子"。

### 核心流程

```
磁盘上的图片 (.jpg/.png)
        ↓
    PIL.Image 读取 (RGB 模式)
        ↓
  [训练时] 随机数据增广
  (水平翻转 / 垂直翻转, 概率各 50%)
  ⚠️ input 和 target 必须用同一个随机种子，否则两张图翻转不同步！
        ↓
  Resize → 统一缩放为 256×256
        ↓
  ToTensor → 像素值从 [0, 255] 变成 [0.0, 1.0]
        ↓
  Normalize(0.5, 0.5) → 像素值从 [0, 1] 变成 [-1, 1]
        ↓
  返回字典 {"input": Tensor, "target": Tensor, "name": 文件名}
```

### 为什么要归一化到 [-1, 1]？
因为我们的网络输出层用的是 **Tanh 激活函数**，Tanh 的值域恰好是 (-1, 1)。输入和输出保持同样的范围，训练时才不会出现尺度错配。

---

## 第二步：神经网络设计 — `model.py`

### 做了什么？
定义了 U-Net 这个"图像处理大脑"的具体结构。

### 为什么选 U-Net？
U-Net 最初为医学图像分割设计，被广泛复用到图像增强任务中。它的关键优势在于 **"跳跃连接 (Skip Connection)"**，能让网络在恢复细节时不丢失低层次的纹理信息。

### 整体结构（像一个 U 形）

```
输入图像 [B, 3, 256, 256]
             │
  ┌──────────────────────────────────────────┐
  │               Encoder 编码器（收缩路径）             │
  │                                          │
  │  enc1: [B, 64, 256, 256] ─────────────skip1─┐  │
  │          ↓ MaxPool                            │  │
  │  enc2: [B, 128, 128, 128] ─────────────skip2─┤  │
  │          ↓ MaxPool                            │  │
  │  enc3: [B, 256, 64, 64]  ─────────────skip3─┤  │
  │          ↓ MaxPool                            │  │
  │  enc4: [B, 512, 32, 32]  ─────────────skip4─┤  │
  │          ↓ MaxPool                            │  │
  │                                          │  │
  │        Bottleneck瓶颈层（最深最小）              │  │
  │  [B, 1024, 16, 16]                       │  │
  │                                          │  │
  │               Decoder 解码器（扩张路径）             │  │
  │          ↑ 转置卷积上采样 + cat(skip4)        │  │
  │  dec4: [B, 512, 32, 32]                  │  │
  │          ↑ 转置卷积上采样 + cat(skip3)        │  │
  │  dec3: [B, 256, 64, 64]                  │  │
  │          ↑ 转置卷积上采样 + cat(skip2)        │  │
  │  dec2: [B, 128, 128, 128]                │  │
  │          ↑ 转置卷积上采样 + cat(skip1)        │  │
  │  dec1: [B, 64, 256, 256]                 │  │
  └──────────────────────────────────────────┘
             │
  1×1 卷积 → Tanh 激活
             │
  输出增强图 [B, 3, 256, 256]（值域 [-1, 1]）
```

### 每一层都在干什么？

| 组件 | 核心操作 | 作用 |
|------|----------|------|
| `DoubleConv` | `Conv3×3 → BN → ReLU` 重复 2 次 | 提取局部特征 |
| `EncoderBlock` | DoubleConv → MaxPool | 空间尺寸减半，通道数加倍，提取更抽象的特征 |
| `Bottleneck` | 仅 DoubleConv，无池化 | 最深层，编码图像的全局语义 |
| `DecoderBlock` | 转置卷积上采样 → cat(skip) → DoubleConv | 逐步恢复空间分辨率，利用跳跃连接补充低层细节 |
| `Output Head` | 1×1 卷积 → Tanh | 将特征通道数从 64 映射回 RGB 3 通道，输出范围 (-1, 1) |

### Skip Connection（跳跃连接）的作用？

> **问题：** 深层特征有"全局理解"但丢失了图像细节；浅层有细节但没有语义理解。
>
> **解决：** 把 Encoder 浅层特征图（保留的 `skip`）通过通道拼接 `cat` 传给 Decoder 对应层，让解码器在恢复细节时同时拥有"语义"和"细节"两种信息。

## 第三步：损失函数设计 — `loss.py`

### 做了什么？
定义一把"尺子"来衡量模型的输出图像和完美参考图究竟有多接近，并通过反向传播告诉模型该怎么改进。

### 论文提出的混合损失公式

$$L_{total} = L_{MSE} + \lambda \cdot L_{SSIM}$$

其中 $\lambda = 1.0$（论文默认设置）

### 为什么要混合两种损失？

| 损失函数 | 公式 | 度量了什么 | 局限性 |
|---------|------|-----------|--------|
| **L_MSE** | `mean((pred - target)²)` | 逐像素的数值误差 | 对平均误差敏感，但不关心图像的结构/纹理 |
| **L_SSIM** | `1 - SSIM(pred, target)` | 亮度、对比度、结构三者的综合相似度 | 单独使用时难以稳定训练 |

二者结合，既保证了**像素数值上的准确性**，又确保了**图像感知质量**（结构和纹理）的还原。

### SSIM 是怎么计算的？
```
对每个 11×11 的局部窗口（高斯加权）：
  1. 计算 pred 和 target 的局部均值 μ₁, μ₂
  2. 计算局部方差 σ₁², σ₂² 和协方差 σ₁₂
  3. 代入公式:
     SSIM = (2μ₁μ₂ + C1)(2σ₁₂ + C2) / ((μ₁² + μ₂² + C1)(σ₁² + σ₂² + C2))
  4. 对整张图所有窗口取平均
```
由于完全使用 PyTorch 原生算子实现，**整个过程可微分，能直接参与反向传播**。

## 第四步：训练流程 — `train.py`

### 做了什么？
把前三个模块串联起来，驱动模型真正"学习"。

### 一次 Epoch（完整训练轮次）的执行流程

```
开始第 N 轮 Epoch
        │
  ┌─────▼──────────────────────────────┐  ← 训练阶段 (model.train())
  │  取出一个 Batch（8张图对）             │
  │  ↓                                 │
  │  前向传播：preds = model(inputs)    │  ← U-Net 处理退化图
  │  ↓                                 │
  │  计算损失：loss = MSE + SSIM        │  ← 用损失函数打分
  │  ↓                                 │
  │  梯度清零：optimizer.zero_grad()    │
  │  ↓                                 │
  │  反向传播：loss.backward()          │  ← 自动计算每个参数该往哪调
  │  ↓                                 │
  │  梯度裁剪：clip_grad_norm(1.0)      │  ← 防止梯度爆炸
  │  ↓                                 │
  │  参数更新：optimizer.step()         │  ← Adam 优化器调整权重
  │  ↓                                 │
  │  重复直到所有 Batch 都跑完            │
  └────────────────────────────────────┘
        │
  ┌─────▼──────────────────────────────┐  ← 验证阶段 (model.eval())
  │  用验证集 178 张跑一遍               │
  │  (torch.no_grad()：只算结果，不更新权重) │
  │  ↓                                 │
  │  记录平均 val_loss                  │
  │  ↓                                 │
  │  保存可视化对比图（每 5 轮一次）       │
  └────────────────────────────────────┘
        │
  学习率自动调整：ReduceLROnPlateau
  (连续 5 轮 val_loss 不下降 → lr × 0.5)
        │
  保存 Checkpoint：
  ├── epoch_xxxx.pth（每 10 轮存一次快照，用于断点续训）
  └── best_model.pth（只要 val_loss 创历史新低就更新）
        │
  进入第 N+1 轮...
```

### 关键设计决策说明

| 设计 | 具体实现 | 原因 |
|------|----------|------|
| 优化器 | Adam, lr=0.001 | 论文指定，Adam 对稀疏梯度自适应较好 |
| 学习率衰减 | ReduceLROnPlateau (patience=5) | 当训练停滞时自动"踩刹车"精细调整 |
| 梯度裁剪 | clip_grad_norm(max_norm=1.0) | 防止梯度爆炸导致模型发散 |
| 权重初始化 | Kaiming Normal | 专为 ReLU 激活设计，避免梯度消失 |
| 保存策略 | best_model 只保存历史最优 | 有效防止过拟合（即使后期 loss 上升也不覆盖最好的结果）|

## 整体数据流总结

```
硬盘图片 (.jpg)
    ↓  dataset.py（读取、增广、归一化）
Tensor [-1, 1] → DataLoader（批量打包，送 GPU）
    ↓  model.py（U-Net 前向传播）
Enhanced Tensor [-1, 1]
    ↓  loss.py（与 target 比较，计算 MSE + SSIM）
Loss 标量
    ↓  train.py（反向传播，Adam 更新权重）
更新后的 U-Net 权重 (.pth)
    ↓  下一轮 Epoch…
最终 best_model.pth（验证集上效果最佳的权重）
```

---

> **一句话总结：**  
> `dataset.py` 负责**喂数据**  →  `model.py` 是**"大脑"**  →  `loss.py` 是**"打分官"**  →  `train.py` 是**"教练"**，负责把三者串起来反复练习直到模型学会怎么把绿色偏色的水下图变成自然色彩。
